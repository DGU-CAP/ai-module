# HTTP 에러율 급증 장애 대응

## 증상
- HTTP 5xx 에러율이 5% 이상으로 증가
- 로그에 Connection refused, Timeout 반복
- 특정 엔드포인트에서 집중적으로 에러 발생
- 의존 서비스 호출 실패

## 원인
- DB 연결 풀 고갈
- 의존 서비스(외부 API, 메시지큐 등) 장애
- 최근 배포된 코드의 버그
- 네트워크 정책 변경으로 인한 연결 차단
- 리소스 부족으로 인한 처리 지연

## 조치
1. 에러 로그에서 구체적인 에러 메시지 확인
2. 의존 서비스 상태 확인 (`kubectl get pods -n <namespace>`)
3. DB 연결 풀 설정 확인 및 상향 조정
4. 최근 배포 이력 확인 후 롤백 검토
5. 서킷브레이커 동작 여부 확인

## 예방
- 의존 서비스에 타임아웃 및 재시도 로직 적용
- DB 연결 풀 모니터링 및 적절한 크기 설정
- 카나리 배포로 신규 코드 점진적 적용
