# 메모리 사용률 급증 장애 대응

## 증상
- 메모리 사용률이 85% 이상으로 지속
- 점진적으로 메모리가 증가하는 패턴 (누수 의심)
- GC 빈도 증가로 인한 성능 저하
- OOMKilled 직전 상태

## 원인
- 메모리 누수 (Connection, Stream 등 미해제)
- 캐시 크기 제한 없음
- 대용량 데이터를 메모리에 전부 로드
- JVM heap 설정 부적절

## 조치
1. `kubectl top pod <pod-name>`으로 메모리 추세 확인
2. GC 로그 분석으로 heap 사용 패턴 확인
3. heap dump 생성 후 메모리 누수 분석
4. memory limit 임시 상향 후 근본 원인 분석
5. 캐시 설정에 최대 크기(max size) 적용

## 예방
- 메모리 사용률 80% 이상 시 알림 설정
- JVM: -Xmx를 컨테이너 limit의 75% 이하로 설정
- 코드 리뷰 시 리소스 해제 여부 확인
